## 当前工程定位

`Colorbar_image` 是 Linux 端彩条图像接收工程骨架。它不替代 `pango_pcie_dma_alloc` 的 PCIe/DMA 验证功能，而是在验证通过的基础上，继续做图像帧接收、保存、校验和后续显示。

当前推荐路线：

```text
1. 先保存一帧 RGB565 raw 文件
2. 抽样校验 8 色彩条
3. 确认数据正确后再做 fb0/DRM/GUI 显示
```

## 新增头文件

`include/colorbar_pcie_rx.h` 定义了 Linux 端彩条接收需要的常量和计划使用的 ioctl：

```text
分辨率          1920 x 1080
格式            RGB565
有效帧大小       4,147,200 bytes
帧标记预留       64 bytes
buffer 数量      4
```

注意：这些 ioctl 是给后续视频接收驱动接口预留的。当前已验证的 `pango_pcie_dma_alloc` 驱动还没有实现这些 ioctl，因此 `--once` 现在会提示驱动暂不支持，这是预期现象。

## 新增命令行工具

`src/pcie_color_rx.c` 当前支持：

```sh
./build/pcie_color_rx --info
./build/pcie_color_rx --validate frame_0000.rgb565
./build/pcie_color_rx --once --output frame_0000.rgb565
```

其中：

```text
--info       打印图像格式、帧大小、buffer 大小等信息
--validate   校验一个已有 RGB565 raw 文件的 8 色彩条抽样点
--once       预留的一帧接收流程，需要后续驱动实现 Colorbar RX ioctl/mmap
```

## 下一步建议

第一步不要做 GUI，先让驱动提供最小视频接收接口：

```text
COLORBAR_IOC_ALLOC_BUFS

COLORBAR_IOC_STOP
COLORBAR_IOC_FREE_BUFS

```

驱动实现后，运行：

```sh
sudo ./build/pcie_color_rx --once --output frame_0000.rgb565
./build/pcie_color_rx --validate frame_0000.rgb565
```

如果 8 个抽样点全部 OK，再进入显示阶段。
